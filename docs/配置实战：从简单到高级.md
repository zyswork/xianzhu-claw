1. 通信架构（混合模式）
┌─────────────────────────────────────────────────────────┐
│ 混合通信模式 (Push + Pull)                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 场景 1：企业管理员更新共享知识库                        │
│ ─────────────────────────────────────────────────────  │
│ 1. 管理员上传文档到企业管理后台                        │
│ 2. 企业管理后台发送 Push 通知到所有本地实例            │
│    └─ 通知内容：知识库已更新，请拉取最新版本          │
│ 3. 本地实例收到通知后，主动 Pull 最新数据             │
│    └─ 只下载有变更的文档（增量更新）                  │
│ 4. 本地实例更新本地缓存                                │
│                                                         │
│ 场景 2：本地实例离线                                    │
│ ─────────────────────────────────────────────────────  │
│ 1. 本地实例离线，无法接收 Push 通知                    │
│ 2. 本地实例上线后，主动 Pull 最新数据                 │
│    └─ 检查版本号，如果有更新则下载                    │
│ 3. 本地实例更新本地缓存                                │
│                                                         │
│ 场景 3：本地实例离线时有多个更新                        │
│ ─────────────────────────────────────────────────────  │
│ 1. 企业管理后台记录所有更新事件                        │
│ 2. 本地实例上线后，Pull 所有未同步的事件              │
│ 3. 本地实例按顺序重放事件，确保数据一致性             │
│                                                         │
└─────────────────────────────────────────────────────────┘
2. 离线模式支持（完全离线）
┌─────────────────────────────────────────────────────────┐
│ 完全离线支持                                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 离线时可以做什么：                                      │
│ ✅ 访问本地对话历史                                     │
│ ✅ 访问本地 Agent 长期记忆                             │
│ ✅ 访问本地知识库                                       │
│ ✅ 访问已缓存的企业共享知识库                           │
│ ✅ 执行本地操作 (文件、命令等)                         │
│ ✅ 创建新的对话和 Agent                                │
│ ❌ 访问企业共享知识库的最新版本                         │
│ ❌ 上报 Token 使用                                      │
│ ❌ 同步 Agent 模板库                                    │
│                                                         │
│ 离线数据同步机制：                                      │
│ 1. 本地队列 - 所有离线时的操作存储在本地队列           │
│ 2. 上线后自动同步 - 连接恢复后自动上报数据             │
│ 3. 冲突解决 - 使用时间戳 + CRDTs 解决冲突             │
│ 4. 事件溯源 - 记录所有变更事件，可重放历史             │
│                                                         │
└─────────────────────────────────────────────────────────┘
3. 数据一致性保证（多层机制）
┌─────────────────────────────────────────────────────────┐
│ 数据一致性保证 (版本号 + 时间戳 + CRDTs + 事件溯源)    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 机制 1：版本号 (Version Number)                        │
│ ─────────────────────────────────────────────────────  │
│ 每个文档有版本号：v1, v2, v3, ...                      │
│ 本地实例比较版本号：                                    │
│   本地版本 < 服务器版本 → 下载更新                     │
│   本地版本 = 服务器版本 → 无需更新                     │
│   本地版本 > 服务器版本 → 冲突（使用 CRDTs 解决）     │
│                                                         │
│ 机制 2：时间戳 (Timestamp)                             │
│ ─────────────────────────────────────────────────────  │
│ 每个文档有最后修改时间：2026-03-15 12:00:00           │
│ 冲突解决：最后修改时间最新的版本获胜                   │
│                                                         │
│ 机制 3：CRDTs (Conflict-free Replicated Data Types)   │
│ ─────────────────────────────────────────────────────  │
│ 对于文本内容，使用 CRDT 算法自动合并冲突              │
│ 例如：Yjs、Automerge 等库                             │
│ 优点：自动解决冲突，无需人工干预                       │
│                                                         │
│ 机制 4��事件溯源 (Event Sourcing)                      │
│ ─────────────────────────────────────────────────────  │
│ 记录所有变更事件：                                      │
│   Event 1: 创建文档 "Company Policy"                  │
│   Event 2: 添加内容 "..."                             │
│   Event 3: 修改内容 "..."                             │
│   Event 4: 删除段落 "..."                             │
│ 本地实例可以重放事件，确保最终一致性                   │
│                                                         │
│ 优先级：版本号 > 时间戳 > CRDTs > 事件溯源             │
│                                                         │
└──────────────

5. 企业共享知识库同步（混合方案）
┌─────────────────────────────────────────────────────────┐
│ 企业共享知识库同步 (热数据本地缓存 + 冷数据远程查询)   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 热数据 (本地缓存)                                       │
│ ─────────────────────────────────────────────────────  │
│ 定义：最近 30 天内被访问过的文档                       │
│ 存储：本地 SQLite + Qdrant                             │
│ 优点：搜索快、离线支持好                               │
│ 缺点：存储占用大                                       │
│                                                         │
│ 冷数据 (远程查询)                                       │
│ ─────────────────────────────────────────────────────  │
│ 定义：超过 30 天未被访问的文档                         │
│ 存储：企业管理后台的 PostgreSQL + Qdrant              │
│ 优点：存储占用小                                       │
│ 缺点：搜索需要远程调用、离线不支持                     │
│                                                         │
│ 同步策略                                                │
│ ─────────────────────────────────────────────────────  │
│ 1. 初始同步：下载最近 30 天的热数据                    │
│ 2. 增量同步：每天同步新增/修改的文档                   │
│ 3. 访问时同步：访问冷数据时，自动下载到本地           │
│ 4. 定期清理：超过 30 天未访问的本地数据自动删除       │
│                                                         │
│ 搜索流程                                                │
│ ─────────────────────────────────────────────────────  │
│ 1. 用户搜索 "销售政策"                                 │
│ 2. 本地 Qdrant 搜索热数据                              │
│ 3. 如果结果不足，远程查询冷数据                        │
│ 4. 合并结果，返回给用户                                │
│ 5. 如果用户访问冷数据，自动下载到本地                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
6. Token 使用监控（本地队列 + 定期上报）
┌─────────────────────────────────────────────────────────┐
│ Token 使用监控 (本地队列 + 定期上报)                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 本地队列机制                                            │
│ ─────────────────────────────────────────────────────  │
│ 每次 LLM 调用后：                                       │
│   1. 记录 Token 使用：{timestamp, model, tokens, cost} │
│   2. 存储到本地队列 (SQLite)                           │
│   3. 不立即上报                                        │
│                                                         │
│ 定期上报机制                                            │
│ ─────────────────────────────────────────────────────  │
│ 每隔 5 分钟（可配置）：                                 │
│   1. 检查本地队列中是否有未上报的数据                  │
│   2. 批量上报到企业管理后台                            │
│   3. 上报成功后，从本地队列删除                        │
│   4. 上报失败，保留在本地队列，稍后重试               │
│                                                         │
│ 离线处理                                                │
│ ─────────────────────────────────────────────────────  │
│ 如果本地实例离线：                                      │
│   1. Token 使用继续记录到本地队列                      │
│   2. 上线后自动上报所有未上报的数据                    │
│   3. 确保数据不丢失                                    │
│                                                         │
│ 优点                                                    │
│ ─────────────────────────────────────────────────────  │
│ ✅ 网络开销小 (批量上报)                               │
│ ✅ 数据准确 (本地队列保证)                             │
│ ✅ 离线支持好 (本地缓存)                               │
│ ✅ 实现简单                                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
7. 安全隔离系统（多层防护）
┌─────────────────────────────────────────────────────────┐
│ 安全隔离系统 (多层防护)                                 │

│   - 只允许执行预定义的命令                             │

│   - 例如：rm -rf /, sudo, etc.                        │
